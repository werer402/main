# Данные CRM — полиграфическая компания

Анонимизированные данные CRM-системы (Bitrix24) компании, специализирующейся на производстве этикеточной продукции: самоклеящаяся этикетка, термоусадочная этикетка (с печатью и без печати).

**Клиенты:** B2B — производители пищевой продукции, бытовой химии, автохимии, фармацевтики, напитков, зоотоваров и т.д.

**География:** Беларусь и Россия.

**Валюта:** BYN (белорусский рубль), RUB (российский рубль), реже EUR.

---

## Файлы

### leads.json — 200 лидов

Потенциальные клиенты на разных стадиях обработки.

| Поле | Описание |
|------|----------|
| `ID` | Уникальный идентификатор лида |
| `TITLE` | Название компании |
| `STATUS_ID` | Статус лида (см. pipeline.json → lead_statuses) |
| `SOURCE_ID` | Источник лида (CALL, EMAIL, WEB, TRADE_SHOW и др.) |
| `ASSIGNED_BY_ID` | ID ответственного менеджера |
| `DATE_CREATE` | Дата создания |
| `DATE_MODIFY` | Дата последнего изменения |
| `DATE_CLOSED` | Дата закрытия (для завершённых лидов) |
| `COMPANY_TITLE` | Название компании (может быть null) |
| `NAME` | Имя контактного лица |
| `LAST_NAME` | Фамилия контактного лица |
| `CURRENCY_ID` | Валюта (BYN) |
| `OPPORTUNITY` | Сумма потенциальной сделки |
| `COMMENTS` | Комментарии менеджера |

**Распределение по статусам:**
- ~38% — «Установление контакта с ЛПР» (активная проработка)
- ~21% — «Новый лид» (не обработан)
- ~12% — «Выявление потребности»
- ~10% — «Коммерческое предложение» (КП отправлено)
- ~7% — «Не используют этикетку» (отказ)
- ~6% — «Размещён заказ» (успех)
- остальные — различные причины отказа

### deals.json — 20 сделок

Сделки на разных стадиях воронки продаж.

| Поле | Описание |
|------|----------|
| `ID` | Уникальный идентификатор сделки |
| `TITLE` | Название (обычно = название компании) |
| `TYPE_ID` | Тип сделки (SALE) |
| `STAGE_ID` | Стадия сделки (см. pipeline.json) |
| `CURRENCY_ID` | Валюта |
| `OPPORTUNITY` | Сумма сделки |
| `LEAD_ID` | ID связанного лида |
| `CATEGORY_ID` | ID воронки (0 = основная, 11 = повторная проработка, 13 = готовая этикетка) |
| `STAGE_SEMANTIC_ID` | Семантика: P (process), S (success), F (failure) |
| `SOURCE_ID` | Источник |
| `BEGINDATE` / `CLOSEDATE` | Период сделки |
| `DATE_CREATE` / `DATE_MODIFY` | Даты создания/изменения |
| `CLOSED` | Y/N — сделка закрыта? |
| `IS_RETURN_CUSTOMER` | Повторный клиент? |
| `IS_REPEATED_APPROACH` | Повторное обращение? |
| `ASSIGNED_BY_ID` | Ответственный менеджер |
| `COMMENTS` | Комментарии |

**Суммы:** от 0 до 600 000 BYN. Крупные сделки — промышленные заказы на этикетку.

### activities.json — 50 активностей

Действия менеджеров: звонки, письма, встречи, задачи.

| Поле | Описание |
|------|----------|
| `ID` | Уникальный идентификатор |
| `OWNER_ID` | ID связанной сущности (лид или сделка) |
| `OWNER_TYPE_ID` | Тип сущности: 1 = лид, 2 = сделка |
| `TYPE_ID` | Тип активности: 1 = встреча, 2 = звонок, 4 = email, 6 = задача/TODO |
| `PROVIDER_ID` | Провайдер (VOXIMPLANT_CALL, CRM_EMAIL, CRM_MEETING, CRM_TODO) |
| `SUBJECT` | Тема |
| `CREATED` / `LAST_UPDATED` | Даты |
| `DEADLINE` | Дедлайн |
| `COMPLETED` | Y/N |
| `STATUS` | Статус: 1 = открыта, 2 = выполнена, 3 = завершена автоматически |
| `RESPONSIBLE_ID` | Ответственный |
| `PRIORITY` | Приоритет: 1 = низкий, 2 = средний, 3 = высокий |
| `DESCRIPTION` | Описание / результат |
| `DIRECTION` | Направление: 0 = нет, 1 = входящий, 2 = исходящий |

### dialogs.json — 30 диалогов

Реалистичные примеры переписки/разговоров между клиентами и сотрудниками компании.

| Поле | Описание |
|------|----------|
| `id` | Номер диалога |
| `client` | Название компании-клиента |
| `channel` | Канал: email / phone |
| `scenario` | Тип сценария |
| `roles` | Роли сотрудников, участвующих в диалоге |
| `messages` | Массив сообщений: `from` (client/sales/economist/technologist/dispatcher/manager), `text` |

**Роли сотрудников:**
- `sales` — Менеджер по продажам (холодные звонки, маркетинговые исследования)
- `economist` — Экономист (расчёт стоимости, КП, скидки, ценообразование)
- `technologist` — Технолог (подбор материалов и технологий, решение технических вопросов)
- `dispatcher` — Диспетчер (отслеживание заказов, логистика, доставка)
- `manager` — Руководитель (решение конфликтов, согласование скидок, претензии)

**Сценарии:**
- Расчёт стоимости и ценовые переговоры
- Подбор материалов и технологий, техническая консультация
- Отслеживание доставки
- Холодный обзвон и онбординг нового клиента
- Обработка рекламаций и претензий
- Согласование макетов
- Тестирование образцов
- Срочные заказы
- Экологические требования
- Премиальная этикетка (тиснение, конгрев, трафаретный лак)
- Малые тиражи
- Мультиязычная продукция

### pipeline.json — Структура CRM

Справочник статусов, воронок и причин отказа.

**Статусы лидов (lead_statuses):**
- 4 статуса процесса: NEW → Контакт с ЛПР → Выявление потребности → КП
- 1 статус успеха: CONVERTED (Размещён заказ)
- 8 статусов отказа: не используют этикетку, работают с посредником, не прошли по ценам/ТЗ/срокам/логистике, банкроты, другое

**Воронки сделок (deal_pipelines):**
- Основная (самоклеящаяся этикетка) — 6 стадий процесса + 6 причин проигрыша
- Повторная проработка — 4 стадии
- Продажа готовой этикетки — 4 стадии

**Причины отказа (rejection_reasons):**
13 детализированных причин для лидов и сделок.

## Как использовать

1. **Анализ воронки:** Загрузите leads.json и pipeline.json, постройте воронку конверсии по статусам
2. **Анализ сделок:** deals.json содержит суммы и стадии — можно считать средний чек, конверсию, причины потерь
3. **Анализ активностей:** activities.json показывает рабочий процесс менеджеров — звонки, письма, задачи
4. **AI-ассистент:** dialogs.json — обучающая выборка для чат-бота, который может помогать в ролях экономиста, технолога, диспетчера, менеджера по продажам
5. **Маршрутизация запросов:** По тексту сообщения клиента определить, к какой роли (sales/economist/technologist/dispatcher/manager) нужно направить обращение

## Терминология

| Термин | Значение |
|--------|----------|
| ЛПР | Лицо, принимающее решение |
| ЛВР | Лицо, влияющее на решение |
| КП | Коммерческое предложение |
| ТЗ | Техническое задание |
| SKU | Stock Keeping Unit — единица учёта товара |
| ПП | Полипропилен (материал этикетки) |
| ПЭ | Полиэтилен |
| ПВХ | Поливинилхлорид |
| ПЕТ | Полиэтилентерефталат гликоль |
| БОПП | Биаксиально-ориентированный полипропилен |
| Адгезив | Клеевой слой этикетки |
| Ламинация | Защитная плёнка поверх печати |
| Тиснение | Нанесение фольги (золото, серебро) горячим штампом или формой |
| Конгрев | Объёмное (рельефное) тиснение горячим штампом без краски |
| Высечка | Вырубка этикетки по контуру |
| Флексопечать | Способ печати (флексография) |
| Цифровая печать | Способ печати цифровыми красками (для коротких тиражей) |
| Офсет | Технология плоской печати изображения |
